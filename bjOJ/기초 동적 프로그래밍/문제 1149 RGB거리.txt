문제 1149 RGB거리


규칙성을 보면
다음 집의 색은  이전 집의 색과 달라야 한다

이것도 점진적으로 집의 최소값을 선택하는 것이므로 dp를 이용 (동적 프로그래밍)

우선 1,2번째 집이 선택은 
입력한 2줄중 최소값을 골라야한다

2줄 각각 합을 구하는데 
포문 이용
이중 포분을 사용하는데 
최솟값을 갱신해줌 갱신 될때 마다 i j 값도 저장한다


이렇게 저장된 i ,j값을 
각각 dp[0], dp[1]에 값을 저장한다
그수 다음 값을 저장하는데 
dp[n]= dp[n-1] + dp[index] 식으로 점진적으로 접근

index는 이전에 선택한 인덱스 값을 피하며 
나머지 인덱스 중 최소값을 선택한다

걱정하는 부분

101 101 100
101 100 101 
101 0    101 


202인데 위 dp 알고리즘에서는 301이 됨
즉 최소값은 전꺼말고도 전전의 값을 고려해야 한다
->위의 방식 틀림
전전을 고려한다는 말= rgb 가 3가지라 3가지를 고려한다는 것 
만약 rgbe라고 하면 4가지 고려해야 함
전, 전전, 전전전 을 고려해야 함


d[n]= d[n-3] + ( n, n-1, n-2 에있는 것중 최솟값을 고려한다는 것이다 ) 
->3중 포문으로 최솟값을 구한다


------------------------------------------------

+놓친부분
점진적인 선택을 한다는 것은 알았으나
2차배열을 만들어서 이전에 선택한 값의 세분화된 
dp를 생각하지 못함

답의 주요 키워드
1. 현재와 이전에 선택한 집의 색깔을 중요시 여김
-->그래서 그것을 따로 색깔별로 dp를 만듦
2. 점진적인 계산
현재 선택되는 색깔은 단지 이전에 선택된 색깔에만 영향을 받음

내가 놓친부분
전의 값은 전전에 영향
전전은 전전전에 영향
전전전은 전전전전에 영향
-->이것을 보면 색깔별로 생각보다는
현재값은 이전의 값만 고려하면 되겠구나라고 생각해야함
+그리고 
전의 값은 3가지로 나누어 진다는 점
-->그래서 이차 배열을 통해 구성

*구성
이차배열로 dp 구성

포문 이용
-입력된 값 바로 저장
-그리고 현재 선택되는 값에 따라(3가지) 이전 가능한 색깔중 최소값을 선택해 입력된 색깔의 집값을 더해줌
